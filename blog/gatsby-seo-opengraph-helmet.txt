<span class="code-line"><span class="line-number">28</span><span class="line-content"></span></span></code></pre><h2>Step 4 - Go further with JSON-LD and Linked data</h2>
<p>So far so great, we have enough here for most social media sites to understand the structure of our data and to use this to correctly format the information on a consuming website. But what do search engines use?</p>
<p>The answer is <a href="https://json-ld.org/">Json-ld and linked data</a>, best explained by the specs website itself,</p>
<blockquote>
<p>JSON-LD is a lightweight Linked Data format. It is easy for humans to read and write. It is based on the already successful JSON format and provides a way to help JSON data interoperate at Web-scale. JSON-LD is an ideal data format for programming environments, REST Web services, and unstructured databases such as Apache CouchDB and MongoDB.</p>
</blockquote>
<p>and,</p>
<blockquote>
<p>Linked Data empowers people that publish and use information on the Web. It is a way to create a network of standards-based, machine-readable data across Web sites. It allows an application to start at one piece of Linked Data, and follow embedded links to other pieces of Linked Data that are hosted on different sites across the Web. </p>
</blockquote>

<p>There are actually various websites we can use to validate your tags and data. When building my website I used the following sources.</p>
<ul>
<li><a href="https://cards-dev.twitter.com/validator">Twitters Card testing tool</a></li>
<li><a href="https://search.google.com/structured-data/testing-tool/u/0/">Googles link for testing json-ld</a>, testing and validating tool for JSON-LD. It goes as far as telling you if there are any missing tags.</li>
<li><a href="https://metatags.io/">Matatags.io</a>, great for previewing how you website links will render on all the social media websites</li>
</ul>
