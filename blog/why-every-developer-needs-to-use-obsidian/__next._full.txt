12:["$","$L15",null,{"url":"https://www.faesel.com/blog/why-every-developer-needs-to-use-obsidian","title":"Why every developer needs to use Obsidian","description":"As a engineer learning new languages, tools frameworks etc is just part and parcel of the job. 16:T43af,<p>As a engineer learning new languages, tools frameworks etc is just part and parcel of the job. Over time the spectrum of knowledge a full stack engineer has to learn has increased dramatically. Maintaining a cognitive memory across all the different CLI&#39;s, languages/frameworks etc in today&#39;s setting is no small feat.</p>
<p>Over time i&#39;ve found that if you want something to stick you have to write it down, and if it doesn&#39;t stick (or un-sticks over time) you need some notes to look back at it. As my brain üß† ages these notes have become invaluable, and the need for quality notes over rushed pasted snippets has become more of a priority.</p>
<p>Due to this I decided to revisit and reassess how I learn, and in doing so I ended up discovering <a href="https://obsidian.md/">Obsidian</a> which has been a game changer for me. This article is an engineers perspective of creating dev notes to aid in learning and recollection.</p>
<h1>My old setup</h1>
<p>Prior to <a href="https://obsidian.md/">Obsidian</a> my note taking activities all occurred in OneNote, I had both my personal notes as well as learning notes around different technologies all in one place split by sections.</p>
<p><img src="//images.ctfassets.net/wjg1udsw901v/o5aNPcsMQrTCGBp8ffbJY/454ecc55a38b36a53093897c1bc32181/OneNote_Structure.png" alt="OneNote Structure"><br><em>My programming category would contain a new section per technology</em></p>
<p>The notes themself&#39;s contained a mixture of,</p>
<ul>
<li>Images (mostly diagrams)</li>
<li>Paragraphs</li>
<li>Code snippets (sure you can use gists or a repo, but i want context with my code snippets and i want it to be searchable)</li>
<li>Links to articles, tools, documentation links ect.</li>
<li>Youtube links usually tutorials</li>
</ul>
<p>However I realized that there are some serious shortcomings with OneNote, and these problems are replicated across its competitors too, like Notion or Evernote. Heres my list,</p>
<ul>
<li>Code pasted in is never pure code, they always add markup on top to render which means copying and pasting out adds characters or unwanted proprietary formatting</li>
<li>Diagrams are always disconnected, hard to edit. They are usually pasted in images from other tools.</li>
<li>I don&#39;t have any version tracking, history or backup.</li>
<li>Pasting and editing is distracting with as styling gets pasted in from the source.</li>
<li>There are no connections in my notes, linking is not possible.</li>
<li>I dont truly own any of the content, i will forever be tied to whatever I write in.</li>
</ul>
<p>The list can go on, but you get the idea...</p>
<h1>How Obsidian makes it better</h1>
<p>Obsidian is part of a next generation in IDE&#39;s designed specifically for research and note taking in their own words they describe it as,</p>
<blockquote>
<p>A second brain, for you, forever. Obsidian is a powerful knowledge base that works on top of a local folder of plain text Markdown files.</p>
</blockquote>
<p>Its the VS code of markdown... no quite literally. At its base it provides a really smooth cross platform writing experience with some key features,</p>
<ul>
<li>Markdown editing, you get all the benefits of being able to easily style without leaving your keyboard, as well as encorperating code snippets easily.</li>
<li>Tagging and creating links (backlinks) across your markdown files</li>
<li>Graph view (more on that later)</li>
<li>Powerful built in search</li>
</ul>
<p>And just like VS code its got a rich plugin ecosystem that&#39;s powered by opensource github projects that greatly extend its usability and allows you to mold the app to your liking.</p>
<h2>You own what you write</h2>
<p>Obsidian uses local markdown files that are stored on your own computers file system. There is no proprietary formatting, encoding etc, what you create is just standard markdown files (.md) and a bunch of folders.</p>
<p><img src="//images.ctfassets.net/wjg1udsw901v/7CdSonGKup7xhgpkBtucs7/5ff5b1baababda93c470bebee41bb464/Obsidian_File_Structure.png" alt="Obsidian File Structure"></p>
<p>Why is this important you might ask?</p>
<ul>
<li>It means the notes you painstakingly curate are not locked into a single echo-system, you can shift between them as long as whatever your shifting to supports markdown.</li>
<li>You also get greater control over how you backup and manage change tracking.</li>
<li>You have greater control over the privacy of what you write, in a world full of breaches a shift away from centralized cloud storage is refreshing. <em>harshibar</em> sums it up nicely with this video,</li>
</ul>
<p><a href="https://www.youtube.com/watch?v=HhWUjp5pD0g">https://www.youtube.com/watch?v=HhWUjp5pD0g</a></p>
<h2>Works with git</h2>
<p>Obsidian doesn&#39;t support git right out the box, it requires a community plugin called <a href="https://github.com/denolehov/obsidian-git">Obsidian Git</a>. However after having installed the plugin you end up with the greatest change tracking/archiving tool at your disposal.</p>
<p><img src="//images.ctfassets.net/wjg1udsw901v/gTn99qeodAoxHw1ybqvZg/a504f3b5e0547c47475d2071fb3ab076/Obsidian_Git.png" alt="Obsidian Git"></p>
<p>Whilst the extension is fairly simple in that its mainly for pushing changes to a selected branch. It offers enough functionality to be out of your way. Dealing with more advanced scenarios like merge conflicts might need to resort to the CLI.</p>
<h2>Mermaid</h2>
<p>As a visual thinker who thinks takes more information in from pictures than words the expression <em>a picture speaks a thousand words</em> really hits home. In the past I would always use free tools like <a href="www/draw.io">draw.io</a> to make diagrams then paste in the exported jpg&#39;s into my notes.</p>
<p>As a process I always found this really disconnected, and its even more jarring when you need to make edits and go through the whole export to jpeg flow all over again. I also find with these tools getting boxes to line up with connecting lines in a way that doesn&#39;t look a mess is incredibly fiddly and time consuming.</p>

<p>Admittedly whilst this looks really cool i haven&#39;t made much use out of this so far. It could be because I always target specific topics to learn and research into them rather than just free write so there&#39;s no real discovery of connections happening. In a team setting however the graph view could be really useful for content discovery especially when its cross cutting (like filtering on any article that&#39;s tagged with infrastructure).</p>

<h1>Themes</h1>
<p>Obsidian also supports a bunch of themes and looks utterly glorious, I use a theme called <em>Atom</em>.</p>

<h1>Conclusion</h1>
<p>Obsidian has really refreshed how I work to a modern standard. Hopefully this has given a good overview of the benefits from a engineers perspective.</p>
<p>The only caveat to Obsidian is the lack of mobile apps (its currently being worked on, check out there roadmap on <a href="https://trello.com/b/Psqfqp7I/obsidian-roadmap">Trello</a>). But to be honest I haven&#39;t really felt the need for it as im always learning whilst on my computer.</p>
<p>Obsidian is free try use (and fully functional with its free tier), they also have some paid tiers and boltons, so do check it out!</p>
